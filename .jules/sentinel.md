## 2026-03-03 - Path Traversal Vulnerability in File Methods
**Vulnerability:** Methods `scan_inventory` and `save_recommendations` in `StockSenseAgent` accepted arbitrary file paths (`inventory_file` and `output_file`) without validating if they were constrained within a specific safe directory boundary (e.g., `data/` or `output/`). This allowed a path traversal attack where an attacker could read or write sensitive files on the system using paths like `../../etc/passwd`.
**Learning:** This existed because file input paths were directly passed to file parsing and saving commands (`pd.read_csv`, `os.makedirs`, `open`) without explicitly normalizing and verifying the path against an expected root directory. The application blindly trusted input paths.
**Prevention:** Always normalize incoming file paths using `os.path.realpath` and enforce directory boundaries using `os.path.commonpath([base_dir, target_path]) == base_dir`. Fail securely (e.g., `raise ValueError` or return `None`) if a path traversal is detected, ensuring that exceptions or stack traces do not expose system information.
