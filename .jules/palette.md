## 2024-03-12 - Resilient Keyboard Focus & Screen Reader Noise
**Learning:** Hardcoding focus outline colors (like a solid blue or gray) creates low-contrast accessibility failures when the app uses varying backgrounds (like the dark purple hero gradient vs. the white features section). Additionally, using raw emojis as structural icons without semantic wrappers causes screen readers to redundantly announce decorative text, adding noise.
**Action:** Use `outline: 3px solid currentColor` for `:focus-visible` globally. The `currentColor` keyword automatically inherits the text color of the element, ensuring the focus ring dynamically contrasts against any background the element itself is visible on. Always wrap structural decorative emojis in an element with `aria-hidden="true"` to hide them from the accessibility tree.
